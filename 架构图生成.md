## 1. 视觉与排版标准 (Visual & Layout Standards)

- **绘图引擎**: Mermaid.js (graph TD)
- **排版核心**：**纵向延伸，严禁横向膨胀**。通过合并琐碎子节点，确保架构图在移动端或窄屏下依然具备极佳的可读性。
- **命名规范 (Standard)**：100% 的节点、容器、连线描述必须遵循 **`中文 (English Name)`** 格式。
- **色彩语义 (CSS Class)**:
    - `classDef agent` fill:#fff9c4,stroke:#fbc02d,stroke-width:2px; (智能体/逻辑节点 - 浅黄)
    - `classDef storage` fill:#e8f5e9,stroke:#4caf50,stroke-width:2px; (存储/缓存/数据库 - 浅绿)
    - `classDef gateway` fill:#ffe0b2,stroke:#fb8c00,stroke-width:2px; (判断/分流网关 - 橙色)
    - `classDef async` fill:#f3e5f5,stroke:#9c27b0,stroke-width:2px; (异步任务/后处理 - 浅紫)

## 2. 交互式降噪与封装约束 (Interactive Constraints)

- **封装原则 (The Container Principle)**:
    - **禁止平铺**：严禁在主图平铺展示超过 3 个同类子组件（如：指标库、维度库、模板库应合并为“语义知识库”）。
    - **细节隐藏**：利用 Mermaid 的 `tooltip` 功能承载二级信息。当鼠标悬浮在封装节点时显示具体内容。
- **标注解耦 (De-coupling)**:
    - **逻辑参数**：复杂的业务参数（如：`Max Iterations = 5`）禁止写在 Agent 节点内部。
    - **操作**：将其作为节点的旁注（Note）或使用小型红色虚线框进行标注。
- **连线说明**: 描述文字严禁超过 **4 个汉字**（不计英文括号内容），确保线条间距整齐。

## 4. 交付物生成指令 (Execution Instruction)

**AI 执行官请严格执行：**
1. **中英文对齐自检**：检查全图 100% 的文本，确保没有任何遗漏英文括号的孤立中文。
2. **容器化重组**：主动合并导致图面过宽的子组件，显著提升视觉排版的“呼吸感”。
3. **输出 HTML 代码**：提供包含 Mermaid 渲染引擎、支持 `zoom: true` 和 `click` 事件的完整 HTML。
4. **添加图例说明 (Legend)**：在页面底部生成视觉图例，解释不同颜色、边框和线段的语义。
