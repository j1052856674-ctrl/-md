# 🏗️ 通用架构图生成协议 (Universal Architecture Protocol) v4.0

> **协议目标**：标准化 AI 绘图逻辑，确保架构图具备：**层级解耦、视觉降噪、闭环透明、交互增强**。

---

## 1. 绘图引擎与视觉标准 (Visual Standards)

- **Engine**: Mermaid.js (graph TD)
- **布局哲学**：**层级解耦 (Subgraphs)**。必须按照业务流向（从上至下或从左至右）划分泳道。
- **色彩语义定义 (CSS Class)**:
    - `classDef agent` fill:#fff9c4,stroke:#fbc02d,stroke-width:2px; (智能体/逻辑节点-浅黄)
    - `classDef storage` fill:#e8f5e9,stroke:#4caf50,stroke-width:2px; (存储/缓存-浅绿)
    - `classDef gateway` fill:#ffe0b2,stroke:#fb8c00,stroke-width:2px; (判断/分流-橙色菱形)
    - `classDef async` fill:#f3e5f5,stroke:#9c27b0,stroke-width:2px; (异步任务/后台分析-浅紫)
- **连线规范**:
    - **粗实线 (Thick Line)**: 核心业务主路径。
    - **细实线 (Thin Line)**: 常规数据流转。
    - **虚线 (Dashed Line)**: 知识注入、后台记录、异步通知。
    - **双向线 (Bidirectional)**: 歧义校验、对齐协商逻辑。

## 2. 交互与展示约束 (UI Constraints)

- **视觉降噪**: 连线描述严禁超过 **4 个汉字**。
- **信息隐藏**: 复杂的参数定义、安全阀数值（如 Max Iterations）必须通过 `tooltip` 挂载在节点上，禁止直接写在图面导致臃肿。
- **HTML 封装**: AI 必须输出包含完整 Mermaid 渲染引擎的 HTML 源码，确保“开箱即用”。

## 4. 架构图生成指令 (Generation Instruction)

**AI 执行官请注意：**
1. 严格遵守上述 `classDef` 样式，确保不同职能的节点颜色统一。
2. 每一个核心 Agent 必须归属于对应的 `subgraph`（泳道）。
3. 检查线条交叉，通过调整节点声明顺序优化布局，确保架构图具备“呼吸感”。
4. 输出完整的 HTML 代码，并在页面底部附带一个**“架构图例 (Legend)”**。
